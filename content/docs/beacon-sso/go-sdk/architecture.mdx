# 架构设计

Beacon SSO SDK 采用分层架构设计，与 bamboo-base-go 框架深度集成。

## 整体架构

```mermaid
graph TB
    subgraph Application[应用层]
        Route[Route 路由注册]
        Handler[Handler 请求处理]
        Middleware[Middleware 中间件]
    end

    subgraph Business[业务层]
        OAuthLogic[OAuthLogic<br/>OAuth 逻辑]
        AuthLogic[AuthLogic<br/>认证逻辑]
        UserLogic[UserLogic<br/>用户逻辑]
        BusinessLogic[BusinessLogic<br/>业务逻辑]
    end

    subgraph Client[客户端层]
        SsoClient[SsoClient<br/>gRPC 客户端]
        OAuth2[OAuth2 Config<br/>HTTP 客户端]
    end

    subgraph Data[数据层]
        Repository[Repository 数据仓储]
        Cache[Redis 缓存]
    end

    subgraph External[外部服务]
        SSOServer[SSO Server<br/>OAuth2 + gRPC]
    end

    Route --> Handler
    Handler --> OAuthLogic
    Handler --> AuthLogic
    Handler --> UserLogic
    Middleware --> OAuthLogic

    OAuthLogic --> OAuth2
    OAuthLogic --> Repository
    AuthLogic --> SsoClient
    UserLogic --> SsoClient
    BusinessLogic --> OAuth2

    SsoClient --> SSOServer
    OAuth2 --> SSOServer
    Repository --> Cache
```

## 依赖注入模式

SDK 采用 bamboo-base-go 的 Register 依赖注入模式：

```go
// startup/startup.go
func NewStartupConfig(exclude ...string) []xRegNode.RegNodeList
```

该函数聚合了以下注册节点：

| 节点名称 | 说明 |
|---------|------|
| `oAuthConfig` | OAuth2 核心配置（ClientID、Endpoint 等） |
| `oAuthRedirectURI` | OAuth2 重定向地址 |
| `ssoClient` | SsoClient gRPC 客户端 |

### 排除节点

你可以通过 `exclude` 参数排除特定节点：

```go
// 排除 gRPC 客户端（仅使用 OAuth2）
nodes := bSdkStartup.NewStartupConfig("ssoClient")

// 排除 OAuth 配置（使用自定义配置）
nodes := bSdkStartup.NewStartupConfig("oAuthConfig", "oAuthRedirectURI")
```

## 核心模块职责

| 模块 | 目录 | 职责 |
|------|------|------|
| **Startup** | `startup/` | 依赖注入节点定义 |
| **Route** | `route/` | Gin 路由注册 |
| **Handler** | `handler/` | HTTP 请求处理 |
| **Logic** | `logic/` | 业务逻辑封装 |
| **Client** | `client/` | gRPC 客户端 |
| **Repository** | `repository/` | 数据仓储层 |
| **Models** | `models/` | 数据模型定义 |
| **Middleware** | `middleware/` | Gin 中间件 |
| **Constant** | `constant/` | 常量定义 |
| **Utility** | `utility/` | 工具函数 |

## 两大子系统

### OAuth2 子系统

基于 `golang.org/x/oauth2` 扩展，提供：

- **授权码流程** - State + PKCE 防护
- **Token 管理** - Redis 缓存、自动刷新
- **业务能力** - Userinfo、Introspection（带缓存）

```mermaid
sequenceDiagram
    participant User
    participant App
    participant SDK as Beacon SSO SDK
    participant SSO as SSO Server
    participant Redis

    User->>App: 访问受保护资源
    App->>SDK: OAuthRouter.OAuthRouter()
    SDK->>SDK: Create() 生成 State + PKCE
    SDK->>Redis: 缓存 State + Verifier
    SDK->>User: 302 重定向到 SSO
    User->>SSO: 授权
    SSO->>User: 重定向回 callback?code=xxx&state=xxx
    User->>SDK: GET /oauth/callback
    SDK->>Redis: 验证 State，获取 Verifier
    SDK->>SSO: Exchange(code, verifier)
    SSO-->>SDK: Access Token + Refresh Token
    SDK->>Redis: 缓存 Token
    SDK-->>User: 登录成功
```

### gRPC 子系统

基于 `connectrpc.com/connect` 实现，提供：

| 服务 | 说明 | 认证要求 |
|------|------|----------|
| **Public** | 公共服务（发送验证码） | 无 |
| **Auth** | 认证服务（注册/登录/改密） | App 凭证 |
| **User** | 用户服务（获取用户信息） | App 凭证 + User Token |
| **Merchant** | 商户服务（标签/公告） | App 凭证 |

## 与 bamboo-base-go 的集成

SDK 依赖 bamboo-base-go 的以下能力：

| 能力 | 用途 |
|------|------|
| `xReg.Register` | 依赖注入容器 |
| `xCtxUtil.MustGetDB` | 从上下文获取数据库 |
| `xCtxUtil.MustGetRDB` | 从上下文获取 Redis |
| `xError` | 统一错误处理 |
| `xLog` | 结构化日志 |
| `xResult` | 统一响应格式 |
